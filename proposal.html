Recommended architecture (production)

CSV / API import → PostgreSQL (contacts, campaigns, states, email history) → Enrichment layer (company URL fetch, CSV fields — no LinkedIn scrape without official API) → Claude / OpenAI (structured JSON: subject, body, personalization rationale) → Review UI (Next.js): Draft → Edit → Approve → Reject → n8n: approved-only send queue → SMTP / Gmail → Inbound webhook/IMAP parser → status = Replied → Scheduler: no reply 14d → generate follow-up → approval gate → send → Dashboard: campaign KPIs + audit log

Why n8n + FastAPI + Next.js

  • n8n — send queues, retries, cron, webhooks; your team can edit flows visually
  • FastAPI — LLM calls, enrichment, state machine, GDPR-safe logging
  • Next.js — approval dashboard your staff actually uses
  • PostgreSQL — single source of truth for campaigns + audit

Compliance (Switzerland / EU)

  • Human approval before every outbound send (default)
  • No automated LinkedIn scraping (ToS + GDPR risk)
  • Enrichment: CSV columns + optional public website text fetch
  • Full audit trail per contact + campaign

Similar work
